# السياق حسب المكان والتذكيرات الذكية

## 1. الهدف

تهدف هذه الميزة إلى جعل النظام يفهم سياق المستخدم حسب المكان، مثل الوصول إلى العمل أو المنزل، ثم يعرض المعلومات والمهام المناسبة لذلك المكان، مع دعم تسجيل الملاحظات والتذكير بها لاحقًا في الوقت أو المكان المناسب.

الفكرة الأساسية: عندما يصل المستخدم إلى العمل، يعرض النظام بيانات العمل وما يجب فعله اليوم. وعندما يصل إلى المنزل، يعرض تذكيرات المنزل أو الأمور الشخصية. كما يمكن للمستخدم أن يقول أو يكتب ملاحظة مثل: “ذكرني بهذا الموضوع لاحقًا”، فيحفظها النظام ويربطها بالسياق المناسب.

## 2. المبادئ الأساسية

- لا يتم تتبع الموقع إلا بموافقة المستخدم.
- لا يتم إرسال بيانات الموقع إلى أي خدمة خارجية دون موافقة صريحة.
- يمكن تعطيل ميزة الموقع في أي وقت.
- لا تظهر بيانات حساسة على شاشة القفل.
- لا يتم عرض بيانات العمل إلا بعد مصادقة مناسبة إذا كانت حساسة.
- يتم استخدام الموقع لتقديم المساعدة فقط، وليس للمراقبة الخفية.

## 3. تعريف الأماكن المهمة

يجب أن يسمح النظام للمستخدم بتعريف أماكن موثوقة مثل:

- العمل.
- المنزل.
- المستودع.
- المكتب.
- فرع معين.
- موقع عميل مهم.

كل مكان يجب أن يحتوي على:

- اسم المكان.
- نوع المكان.
- نطاق جغرافي تقريبي.
- الإجراءات المرتبطة به.
- مستوى الخصوصية المطلوب.

مثال:

```text
اسم المكان: العمل
النوع: مكتب / مستودع
عند الوصول: عرض لوحة عمل اليوم
عند المغادرة: عرض ملخص اليوم أو تنبيهات غير منجزة
الخصوصية: عالية
```

## 4. عند الوصول إلى العمل

عند اكتشاف أن المستخدم وصل إلى العمل، يمكن للنظام أن يعرض لوحة يومية تحتوي على:

- مهام اليوم.
- تقرير مختصر عن المخزون.
- تنبيهات العملاء المتأخرين عن الدفع.
- فواتير الشراء التي تحتاج طباعة أو أرشفة.
- نشرة الأسعار التي يجب تحديثها.
- سعر الصرف المطلوب لإصدار نشرة الليرة السورية.
- الرسائل أو المنشورات المقترحة للزبائن.
- الفواتير أو التقارير التي تحتاج مراجعة.
- أي ملاحظات قال المستخدم إنه يريد تذكرها في العمل.

## 5. مثال لوحة الوصول إلى العمل

```text
صباح الخير، وصلت إلى العمل.

ملخص اليوم:
1. يوجد 4 عملاء يحتاجون متابعة دفع.
2. نشرة الأسعار تحتاج تحديث سعر الصرف.
3. يوجد 3 أصناف نفدت من المستودع ويجب حذفها من النشرة.
4. توجد فاتورة شراء جديدة تحتاج طباعة للأرشيف.
5. لديك ملاحظة محفوظة: راجع طلب العميل أحمد قبل الظهر.

هل تريد فتح لوحة العمل؟
```

## 6. عند مغادرة العمل

عند مغادرة العمل، يمكن للنظام عرض ملخص أو تذكير مثل:

- هل تم إنشاء تقرير نهاية اليوم؟
- هل تم طباعة فواتير الشراء؟
- هل تم إرسال نشرة الأسعار؟
- هل بقيت مهام غير منجزة؟
- هل توجد ملاحظات يجب نقلها لليوم التالي؟

مثال:

```text
يبدو أنك غادرت العمل.
قبل إنهاء اليوم:
- تقرير نهاية اليوم لم يتم مراجعته بعد.
- هناك عميلان يحتاجان متابعة غدًا.
- توجد فاتورة شراء لم تُطبع للأرشيف.
```

## 7. عند الوصول إلى المنزل

عند الوصول إلى المنزل، يجب أن يتغير السياق من العمل إلى الوضع الشخصي أو الهادئ.

يمكن للنظام عرض:

- تذكيرات شخصية.
- مهام منزلية.
- ملاحظات قال المستخدم إنه يريد تذكرها في المنزل.
- تنبيهات غير حساسة.
- ملخص مختصر جدًا للعمل إذا اختار المستخدم ذلك.

لا يجب عرض بيانات مالية أو أسماء عملاء أو تفاصيل عمل حساسة تلقائيًا عند المنزل إلا إذا وافق المستخدم.

## 8. الملاحظات الذكية

يجب أن يستطيع المستخدم تسجيل ملاحظة بسرعة، مثل:

```text
ذكرني عندما أصل إلى العمل أن أراجع فاتورة المورد.
```

أو:

```text
ذكرني في المنزل أن أراجع هذا الموضوع.
```

أو:

```text
ذكرني غدًا صباحًا أن أتصل بالزبون فلان.
```

يجب أن يحلل النظام الملاحظة ويستخرج:

- نص التذكير.
- المكان المرتبط إن وجد.
- الوقت المرتبط إن وجد.
- مستوى الحساسية.
- هل يحتاج مصادقة قبل عرض التفاصيل.

## 9. أنواع التذكيرات

### 9.1 تذكير حسب المكان

مثال:

```text
عند الوصول إلى العمل: راجع نشرة الأسعار.
```

### 9.2 تذكير حسب الوقت

مثال:

```text
غدًا الساعة 10 صباحًا: اتصل بالعميل فلان.
```

### 9.3 تذكير حسب الحدث

مثال:

```text
عند تسجيل فاتورة شراء: ذكّرني بطباعتها.
```

### 9.4 تذكير حسب السياق

مثال:

```text
عندما أفتح لوحة المحاسبة: ذكّرني بمراجعة العملاء المتأخرين.
```

## 10. حماية الخصوصية في التذكيرات

بعض التذكيرات قد تحتوي على أسماء عملاء أو مبالغ أو بيانات حساسة. لذلك يجب:

- عدم عرض التفاصيل الحساسة على شاشة القفل.
- استخدام إشعار عام مثل: “لديك تذكير عمل يحتاج مراجعة”.
- طلب Face ID أو Passkey قبل عرض التفاصيل الحساسة.
- عدم إرسال التذكيرات إلى Apple Watch إلا إذا اختار المستخدم ذلك صراحة، وبشكل غير حساس.
- عدم إرسال بيانات الموقع أو التذكيرات إلى خدمات خارجية دون موافقة.

## 11. عدم التتبع الخفي

ميزة الموقع يجب أن تكون واضحة للمستخدم.

يجب أن يعرف المستخدم:

- أن ميزة الوصول إلى العمل أو المنزل مفعّلة.
- ما الأماكن المسجلة.
- ما الذي يحدث عند الوصول أو المغادرة.
- كيف يوقف الميزة.
- هل يتم حفظ سجل وصول أو لا.

## 12. سجل الأحداث

يمكن تسجيل أحداث محدودة مثل:

- تم تفعيل تذكير عند الوصول إلى العمل.
- تم عرض لوحة العمل.
- تم تأجيل تذكير.
- تم إنجاز تذكير.

ولا يجب تخزين سجل تنقلات كامل أو تفصيلي إلا إذا طلب المستخدم ذلك صراحة، وبسبب واضح.

## 13. إعدادات قابلة للتخصيص

يجب أن يستطيع المستخدم ضبط:

- تفعيل أو تعطيل التذكيرات حسب الموقع.
- تحديد مواقع العمل والمنزل.
- نوع المعلومات التي تظهر عند الوصول.
- هل تظهر تفاصيل العمل في المنزل أم لا.
- هل تحتاج التذكيرات الحساسة إلى Face ID.
- أوقات عدم الإزعاج.
- مدة تأخير التذكير.
- هل يتم تسجيل سجل وصول مختصر أم لا.

## 14. عدم تنفيذ إجراءات تلقائية

عند الوصول إلى مكان معين، النظام يمكنه عرض واقتراح وتذكير، لكنه لا ينفذ إجراءات حساسة تلقائيًا.

لا يقوم تلقائيًا بـ:

- إرسال رسائل للزبائن.
- إصدار نشرة أسعار.
- تعديل بيانات محاسبية.
- طباعة فواتير.
- فتح تقارير حساسة.
- مشاركة الموقع مع أي طرف.

كل إجراء تنفيذي يحتاج موافقة المستخدم.

## 15. مثال سير عمل عند الوصول إلى العمل

```text
1. المستخدم يصل إلى العمل.
2. النظام يكتشف أنه داخل نطاق العمل الموثوق.
3. يعرض إشعارًا عامًا: “لوحة العمل جاهزة”.
4. المستخدم يفتح الإشعار.
5. النظام يطلب Face ID إذا كانت اللوحة تحتوي بيانات حساسة.
6. بعد المصادقة يعرض مهام اليوم والتقارير والتنبيهات.
7. المستخدم يختار ما يريد تنفيذه.
```

## 16. مثال سير عمل لتذكير مسجل بالصوت أو النص

```text
1. المستخدم يقول: ذكرني عندما أصل إلى العمل أن أراجع فاتورة المورد.
2. النظام يحفظ التذكير ويربطه بمكان العمل.
3. عند الوصول إلى العمل، يظهر إشعار عام.
4. بعد المصادقة، يعرض النظام نص التذكير.
5. المستخدم يحدد: تم / تأجيل / تعديل.
```

## 17. النتيجة المطلوبة

النظام يجب أن يعرف السياق المناسب للمستخدم حسب المكان والوقت، فيعرض معلومات العمل عند العمل، وتذكيرات المنزل عند المنزل، ويحول الملاحظات السريعة إلى تذكيرات ذكية، مع حماية الخصوصية وعدم تنفيذ أي إجراء حساس دون موافقة واضحة.
