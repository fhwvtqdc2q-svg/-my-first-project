# ميزات مساعدة الذكاء الاصطناعي لتطبيقات السوشيال ميديا

## 1. الهدف

يحدد هذا المستند جميع وسائل المساعدة التي يمكن أن يقدمها النظام للمستخدم في تطبيقات السوشيال ميديا، مع الالتزام بعدم الدخول المباشر إلى الحسابات، وعدم تغيير أي ميزة أو إعداد دون تحقق من صحة المعلومات وموافقة المستخدم شخصيًا.

النظام يساعد في الفهم، التحليل، الاقتراح، الصياغة، التنبيه، والمراجعة، لكنه لا ينشر ولا يرسل ولا يغير إعدادات الحسابات تلقائيًا.

## 2. القاعدة الأساسية

لا يتم تغيير أي ميزة أو إعداد في تطبيقات السوشيال ميديا إلا بعد تحقق الشروط التالية:

1. التأكد من صحة المعلومة من مصدر موثوق أو من داخل إعدادات التطبيق التي يراها المستخدم.
2. شرح أثر التغيير للمستخدم بلغة واضحة.
3. توضيح المخاطر والفوائد.
4. طلب موافقة المستخدم شخصيًا.
5. استخدام مصادقة قوية عند الحاجة، مثل Face ID أو Passkey أو تطبيق مصادقة.
6. تسجيل القرار في سجل التدقيق دون حفظ أسرار أو كلمات مرور.

## 3. حدود التعامل مع تطبيقات السوشيال ميديا

### مسموح

- شرح إعدادات الخصوصية.
- اقتراح إعدادات أفضل بناءً على تفضيلات المستخدم.
- تجهيز منشورات ورسائل قبل النشر.
- مراجعة النصوص قبل الإرسال.
- تنبيه المستخدم إلى المحتوى الحساس أو غير المناسب.
- تلخيص السياسات والشروط التي يوفرها المستخدم أو تكون متاحة رسميًا.
- إعداد قائمة مهام لمراجعة الحسابات.
- تذكير المستخدم بالميزات الجديدة أو التغييرات المهمة بعد التحقق منها.

### غير مسموح

- تسجيل الدخول إلى الحسابات دون موافقة صريحة وبطريقة رسمية.
- تجاوز حماية التطبيقات أو شروط الاستخدام.
- نشر محتوى تلقائيًا دون مراجعة وموافقة.
- إرسال رسائل خاصة أو جماعية تلقائيًا دون موافقة.
- قراءة رسائل خاصة دون إذن واضح.
- تغيير إعدادات الحساب دون موافقة المستخدم.
- استخدام أدوات خفية للسحب أو النشر أو التتبع.

## 4. أنواع المساعدة الممكنة

### 4.1 مساعدة إنشاء المحتوى

يمكن للنظام مساعدة المستخدم في:

- كتابة منشورات.
- كتابة إعلانات.
- كتابة رسائل للزبائن.
- تحسين الصياغة.
- تصحيح الأخطاء اللغوية.
- تغيير النبرة: رسمية، ودية، تجارية، مختصرة.
- اقتراح عناوين جذابة.
- تجهيز وصف منتج.
- إعداد نص مناسب لجروب واتساب أو فيسبوك.

### 4.2 مراجعة المحتوى قبل النشر

يمكن للنظام مراجعة المحتوى من حيث:

- الوضوح.
- النبرة.
- الأخطاء اللغوية.
- المعلومات الناقصة.
- وجود بيانات حساسة لا يجب نشرها.
- احتمال سوء الفهم.
- توافق النص مع الهدف التجاري.

### 4.3 اقتراحات تجارية

يمكن للنظام اقتراح:

- أي صنف مناسب للإعلان عنه اليوم.
- أي زبائن قد يحتاجون متابعة.
- أي منشورات مناسبة حسب توفر البضاعة.
- رسائل تذكير مهذبة.
- نصوص عروض خاصة.
- محتوى يومي أو أسبوعي.

### 4.4 إدارة النشر دون تنفيذ تلقائي

يمكن للنظام تجهيز خطة نشر، مثل:

```text
اليوم:
- منشور توفر بضاعة جديدة.
- تذكير بنشرة الأسعار.
- رسالة متابعة للزبائن المهتمين.
```

لكن لا يتم النشر إلا بعد موافقة المستخدم.

### 4.5 مراقبة الميزات الجديدة

يمكن للنظام تنبيه المستخدم عند وجود ميزة جديدة أو تغيير مهم في التطبيق، بشرط أن تكون المعلومة مؤكدة.

أمثلة:

- ميزة خصوصية جديدة.
- تغيير في إعدادات الحساب.
- تحديث في شروط الاستخدام.
- تغيير في إعدادات الرسائل أو الجروبات.
- ميزة جديدة للإعلانات أو الحسابات التجارية.

## 5. سياسة التحقق من صحة المعلومات

قبل إخبار المستخدم بميزة جديدة أو تغيير في تطبيق، يجب تصنيف المعلومة كالتالي:

### مؤكدة

المعلومة مأخوذة من مصدر رسمي، مثل:

- صفحة الدعم الرسمية للتطبيق.
- إعداد ظاهر داخل التطبيق.
- إشعار رسمي من التطبيق.
- سياسة أو شروط استخدام رسمية.

### تحتاج مراجعة

المعلومة غير مؤكدة بالكامل أو مصدرها غير رسمي.

في هذه الحالة يجب على النظام أن يقول:

```text
هذه المعلومة تحتاج تأكيدًا من المصدر الرسمي قبل اتخاذ أي إجراء.
```

### غير موثوقة

لا يتم الاعتماد عليها ولا يتم تنفيذ أي إجراء بناءً عليها.

## 6. عدم تغيير الميزات دون موافقة

أي تغيير في إعدادات السوشيال ميديا يجب أن يمر عبر بطاقة موافقة.

مثال:

```text
الإجراء المقترح: تعديل إعداد خصوصية المنشورات.
السبب: تقليل ظهور المحتوى لغير المتابعين.
الأثر: قد يقل وصول المنشورات العامة.
المصدر: إعداد ظاهر داخل التطبيق.
مستوى الخطورة: متوسط.
هل توافق على تنفيذ الخطوات يدويًا؟ نعم / لا
```

## 7. المصادقة عند الإجراءات الحساسة

يجب طلب Face ID أو Passkey أو تطبيق مصادقة قبل:

- نشر محتوى تجاري مهم.
- إرسال رسالة جماعية.
- تعديل إعداد خصوصية.
- ربط حساب بخدمة خارجية.
- مشاركة تقرير أو ملف.
- تغيير إعدادات الجروبات أو الصفحات.

لا يتم الاعتماد على SMS أو رقم الهاتف كوسيلة تحقق أساسية.

## 8. عدم الدخول المباشر إلى الحسابات

الأصل أن النظام لا يدخل مباشرة إلى حسابات السوشيال ميديا. يتم تقديم المساعدة عبر:

- خطوات يدوية واضحة.
- نصوص جاهزة للمراجعة.
- قوائم تدقيق.
- تنبيهات.
- شرح الإعدادات.

إذا احتاج أي تكامل مستقبلي، يجب أن يكون عبر API رسمي وبموافقة صريحة، وبصلاحيات محدودة، وقابلًا للإلغاء.

## 9. حماية التفضيلات الشخصية

يستطيع النظام حفظ تفضيلات المستخدم لتحسين المساعدة، مثل:

- أسلوب الكتابة المفضل.
- الكلمات التي لا يحب استخدامها.
- نوع الإعلانات المفضلة.
- المنتجات الأكثر أهمية.
- قنوات النشر المفضلة.
- أوقات النشر المفضلة.

لكن يجب أن تكون هذه التفضيلات:

- قابلة للمراجعة.
- قابلة للتعديل.
- قابلة للحذف.
- غير مستخدمة للنشر أو الإرسال دون موافقة.

## 10. تنبيه المحتوى الحساس

قبل النشر، يجب أن ينبه النظام إذا احتوى النص على:

- أسعار غير مؤكدة.
- كمية غير متوفرة.
- معلومات داخلية.
- بيانات زبائن.
- وعد تجاري غير مؤكد.
- كلمات قد تسبب سوء فهم.

## 11. ارتباط السوشيال ميديا بالمحاسبة والمخزون

يمكن للنظام أن يساعد في إنشاء محتوى للسوشيال ميديا بناءً على:

- توفر البضاعة.
- نشرة الأسعار اليومية.
- فواتير الشراء الجديدة.
- الأصناف التي عادت للمخزون.
- الأصناف التي لا يجب الإعلان عنها بسبب نفادها.

لكن لا يتم نشر أي شيء إلا بعد مراجعة المستخدم.

## 12. نموذج سير عمل آمن

```text
1. النظام يلاحظ توفر صنف جديد في المخزون.
2. يقترح منشورًا لجروب الزبائن.
3. يتحقق أن الصنف موجود فعليًا وسعره محدث.
4. يعرض النص للمستخدم.
5. يطلب موافقة المستخدم.
6. عند الحاجة يطلب Face ID أو Passkey.
7. المستخدم ينشر يدويًا أو يوافق على الإرسال عبر قناة رسمية.
8. يتم تسجيل العملية في سجل التدقيق.
```

## 13. سجل التدقيق

يجب تسجيل:

- وقت الاقتراح.
- نوع المحتوى.
- مصدر البيانات.
- هل تم التحقق من السعر أو المخزون.
- هل وافق المستخدم.
- هل تم النشر أو الإرسال.

ولا يجب تسجيل:

- كلمات المرور.
- رموز التحقق.
- محتوى الرسائل الخاصة غير المرتبط بالعمل.
- مفاتيح الوصول.

## 14. النتيجة المطلوبة

النظام يجب أن يفعّل كل وسائل المساعدة الآمنة في السوشيال ميديا: إنشاء المحتوى، مراجعة النصوص، التنبيهات، اقتراح المنشورات، متابعة الميزات الجديدة، وربط المحتوى بالمخزون والمحاسبة، مع منع أي تغيير أو نشر أو إرسال دون التحقق من صحة المعلومات وموافقة المستخدم شخصيًا.
