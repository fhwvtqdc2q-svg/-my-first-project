# خطة إدخال ملفات Excel الحقيقية بأمان

## 1. الهدف

بعد نجاح تشغيل النموذج المحلي على اللابتوب، تصبح الخطوة التالية هي استخدام بيانات العمل الحقيقية من ملفات Excel، لكن بطريقة آمنة لا تعدّل الملفات الأصلية ولا تخاطر بالبيانات.

## 2. الملفات المحتملة على جهاز المستخدم

من الملفات التي ظهرت على سطح المكتب:

```text
دفتر الأستاذ.xlsx
دليل المواد.xlsx
```

وقد توجد ملفات أخرى لاحقًا مثل:

```text
ملف الجرد
ملف الفواتير
ملف العملاء
ملف المدفوعات
نشرة الأسعار
```

## 3. القاعدة الأساسية

لا يتم تعديل ملفات Excel الأصلية إطلاقًا.

النظام يجب أن:

1. يقرأ الملف الأصلي.
2. ينشئ نسخة CSV داخل مجلد وسيط.
3. يستخدم نسخة CSV في التقارير.
4. يترك ملف Excel الأصلي كما هو.

## 4. المجلدات المقترحة

```text
input/
  excel/
    دفتر الأستاذ.xlsx
    دليل المواد.xlsx

converted/
  customers.csv
  products.csv
  inventory.csv
  invoices.csv
  payments.csv
```

## 5. خطوات العمل الآمنة

### الخطوة 1: نسخ ملفات Excel إلى مجلد input

يتم وضع نسخة من ملفات Excel داخل:

```text
input/excel/
```

ولا يتم العمل مباشرة على النسخة الوحيدة أو الأصلية.

### الخطوة 2: فحص أسماء الأوراق والأعمدة

قبل التحويل، يجب أن يقرأ النظام:

- أسماء أوراق Excel.
- أسماء الأعمدة.
- عدد الصفوف.
- وجود أعمدة مطلوبة أو ناقصة.

### الخطوة 3: مطابقة الأعمدة

لأن أسماء الأعمدة في ملفات المحاسبة قد تكون عربية أو مختلفة، يحتاج النظام إلى جدول مطابقة.

مثال:

```text
اسم المادة       -> product_name
كود المادة       -> product_code
الكمية           -> available_quantity
السعر            -> price_usd
اسم الزبون       -> customer_name
الرصيد           -> amount_due
تاريخ الفاتورة   -> invoice_date
```

### الخطوة 4: إنشاء CSV وسيط

بعد المطابقة، ينشئ النظام ملفات CSV موحدة الشكل.

### الخطوة 5: تشغيل التقارير على CSV فقط

كل السكربتات الحالية تستخدم CSV، لذلك يتم تشغيلها على النسخ المحوّلة وليس على Excel مباشرة.

## 6. حماية البيانات

يجب الالتزام بالآتي:

- عدم رفع ملفات Excel إلى أي خدمة خارجية.
- عدم إرسالها إلى GitHub.
- عدم تخزينها في المستودع العام.
- عدم وضع بيانات العملاء الحقيقية داخل `samples/` إذا كان المستودع عامًا.
- استخدام مجلد محلي خاص للبيانات الحقيقية.

## 7. تحذير مهم عن GitHub

يجب عدم رفع ملفات العمل الحقيقية إلى GitHub، خصوصًا:

- أسماء العملاء.
- أرصدة العملاء.
- فواتير.
- أسعار شراء.
- بيانات موردين.
- ملفات محاسبة.

لذلك يجب أن يكون مجلد البيانات الحقيقية مستثنى في `.gitignore`.

## 8. المطلوب تنفيذه برمجيًا

إنشاء سكربت:

```text
scripts/inspect_excel_file.py
```

وظيفته:

- قراءة ملف Excel.
- عرض أسماء الأوراق.
- عرض أسماء الأعمدة.
- عرض أول 5 صفوف بدون تعديل الملف.
- إنشاء تقرير فحص داخل `output/`.

ثم لاحقًا إنشاء سكربت:

```text
scripts/convert_excel_to_csv.py
```

وظيفته:

- استخدام خريطة أعمدة.
- تحويل بيانات Excel إلى CSV موحد.
- وضع النتيجة داخل `converted/`.
- عدم تعديل Excel الأصلي.

## 9. النتيجة المطلوبة

إدخال بيانات المستخدم الحقيقية إلى النظام يجب أن يتم تدريجيًا وبحذر:

1. فحص الملف.
2. فهم الأعمدة.
3. مطابقة الأعمدة.
4. تحويل نسخة إلى CSV.
5. تشغيل التقارير.
6. مراجعة النتائج قبل الاعتماد.
