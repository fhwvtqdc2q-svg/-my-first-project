# رؤية المشروع: نظام مساعد ذكي للحماية والتحكم الشخصي

## الفكرة العامة

يهدف المشروع إلى بناء نظام مساعد ذكي مخصص لصاحب الأجهزة، يساعده في إدارة أجهزته وشبكته وتطبيقاته اليومية بطريقة آمنة وواضحة، مع التركيز على الحماية، الخصوصية، سهولة الاستخدام، وعدم تنفيذ أي إجراء مؤثر دون موافقة صريحة من المستخدم.

الفكرة الأساسية هي أن يكون كل شيء تحت تحكم المستخدم، من خلال مساعد ذكي يوضح الخيارات، يقترح الحلول، ينبه إلى المخاطر، ويساعد في تنفيذ الإعدادات المسموح بها دون تجاوز سياسات التطبيقات أو شروط الاستخدام.

## المبادئ الأساسية

1. موافقة المستخدم أولًا

لا يقوم النظام بأي تعديل أو إجراء مؤثر على الجهاز أو الحسابات أو التطبيقات دون الرجوع إلى المستخدم وطلب موافقته الواضحة.

2. الشفافية الكاملة

يجب أن يوضح النظام للمستخدم ما الذي سيحدث، ولماذا، وما أثره المتوقع، وما البيانات التي قد تُستخدم أو تُقرأ قبل تنفيذ أي إجراء.

3. الحد الأدنى من الصلاحيات

لا يطلب النظام أي صلاحية إلا إذا كانت ضرورية لوظيفة واضحة ومباشرة. أي صلاحية إضافية يجب تبريرها وتوثيقها.

4. الخصوصية وعدم التتبع غير المبرر

لا يستخدم النظام أي آليات تتبع أو اتصالات خارجية غير ضرورية. أي اتصال خارجي يجب أن يكون واضحًا ومبررًا وقابلًا للتعطيل عند الإمكان.

5. الالتزام بالقوانين والسياسات

لا يهدف النظام إلى تجاوز سياسات فيسبوك أو إنستجرام أو واتساب أو جوجل أو أي منصة أخرى. الهدف هو مساعدة المستخدم في الإعدادات والاستخدام المشروع والمسموح فقط.

6. عدم السيطرة على حسابات أو تطبيقات الغير

النظام مصمم لمساعدة المستخدم داخل أجهزته وحساباته الخاصة فقط، ولا يستخدم للوصول غير المصرح به أو تغيير إعدادات لا يملك المستخدم حق التحكم بها.

## نطاق النظام المقترح

### حماية الأجهزة

- مراجعة إعدادات الخصوصية والأمان.
- التنبيه إلى الإعدادات الضعيفة أو غير الضرورية.
- اقتراح خطوات تقوية الجهاز دون تعطيل وظائفه الأساسية.
- منع تخزين الأسرار أو المفاتيح داخل ملفات عامة.

### حماية الشبكة

- تقديم إرشادات لتقوية شبكة المنزل أو العمل.
- توضيح الأجهزة المتصلة عند توفر طريقة مشروعة ومصرح بها.
- اقتراح إعدادات آمنة لكلمات المرور، التحديثات، التشفير، وعزل الأجهزة عند الحاجة.
- عدم تنفيذ أي فحص هجومي أو غير مصرح به.

### المساعد اليومي

- تنظيم المهام اليومية.
- اقتراح أفكار وخطط عمل.
- تلخيص المعلومات التي يطلبها المستخدم.
- إعداد نصوص ورسائل وتقارير.
- مساعدة المستخدم في فهم الإعدادات والسياسات بلغة واضحة.

### التطبيقات اليومية

يمكن للنظام مساعدة المستخدم في تطبيقات مثل فيسبوك، إنستجرام، واتساب، وجوجل من خلال:

- شرح الإعدادات المتاحة داخل التطبيق.
- اقتراح إعدادات خصوصية مناسبة.
- إعداد خطوات يدوية واضحة للمستخدم.
- مساعدة المستخدم في صياغة الرسائل والمحتوى.
- توضيح السياسات والشروط بلغة أبسط.

ولا يشمل ذلك:

- تجاوز سياسات المنصات.
- كسر القيود التقنية أو الأمنية.
- الوصول إلى حسابات بدون إذن.
- جمع بيانات الآخرين دون موافقة.
- تشغيل أدوات تتبع أو مراقبة غير معلنة.

## نموذج الموافقة قبل التنفيذ

قبل أي إجراء مؤثر، يجب أن يعرض النظام للمستخدم:

- اسم الإجراء.
- سبب الحاجة إليه.
- البيانات أو الصلاحيات المطلوبة.
- الأثر المتوقع.
- إمكانية التراجع إن وجدت.
- خيار الموافقة أو الرفض.

مثال:

```text
الإجراء المقترح: تعديل إعدادات الخصوصية في التطبيق.
السبب: تقليل ظهور البيانات الشخصية للعامة.
الصلاحيات المطلوبة: لا توجد صلاحيات إضافية، سيتم إرشادك يدويًا فقط.
الأثر: تحسين الخصوصية دون حذف الحساب أو تعطيل التطبيق.
هل توافق على المتابعة؟ نعم / لا
```

## ما يجب تجنبه

- تخزين كلمات المرور أو مفاتيح API داخل المستودع.
- إضافة مكتبات غير ضرورية.
- تشغيل سكربتات تلقائية عند التثبيت دون توثيق.
- إرسال بيانات المستخدم إلى خدمات خارجية دون توضيح وموافقة.
- استخدام صلاحيات واسعة لا تخدم وظيفة محددة.
- الالتفاف على شروط استخدام التطبيقات أو المنصات.

## النتيجة المطلوبة

الهدف النهائي هو بناء نظام مساعد شخصي آمن، واضح، ومفيد، يجعل المستخدم قادرًا على إدارة أجهزته وتطبيقاته ومعلوماته اليومية بسهولة، مع الحفاظ على الخصوصية، احترام السياسات، ومنع أي إجراء غير مفهوم أو غير مصرح به.
