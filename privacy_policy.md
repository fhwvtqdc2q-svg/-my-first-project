# Privacy Policy

Last updated: 2026/05/12

This Privacy Policy explains how this application collects, uses, stores, and protects user information when users use account security, app management, Face ID, two-factor authentication, and AI‑powered features.

This application is designed to help the user manage and protect their own phone, account, connected apps, and security settings.

## 1. Information We Collect

We may collect only the information necessary to provide and secure the application, including:

* Name
* Email address
* Profile picture
* User ID
* Account authentication status
* Account security settings
* Connected apps or services added by the user
* App management preferences
* Security alerts or login activity
* Basic device security information
* User‑provided prompts, questions, or commands used with AI features
* **We do not collect phone numbers.**

If the application uses Google Sign‑In or Google APIs, we may receive basic Google account information authorized by the user, such as name, email address, profile picture, and Google User ID.

## 2. How We Use Information

We use collected information to:

* Sign users in securely
* Manage user accounts
* Protect account access
* Enable Face ID or biometric unlocking
* Enable two‑factor authentication
* Manage connected apps or services
* Provide AI‑powered assistance for the user's own phone and account
* Detect suspicious login or security activity
* Provide security recommendations
* Improve application security and reliability
* Provide customer support
* Comply with legal obligations

We do not request or store users' phone numbers for authentication or any other purpose. All authentication is performed using email-based credentials, device-based biometric methods (such as Face ID) or other secure two‑factor methods that do not rely on SMS or phone numbers.

We do not use user data for purposes unrelated to the application’s security, account, device, or app management features.

## 3. AI‑Powered Features

The application may include AI‑powered features to help the user understand, manage, and protect their own phone, account, connected apps, and security settings.

AI features may help with:

* Explaining account security status
* Suggesting stronger security settings
* Helping manage connected apps or services
* Detecting possible security risks
* Explaining alerts or login activity
* Providing recommendations based on the user’s own device and account settings
* Answering user questions inside the application

AI features may analyze limited information from the user’s own phone or account only when needed for app functionality and when permitted by the user.

The AI features are not used to monitor other people, access other accounts, or collect information from devices that do not belong to the user.

The application does not use AI to secretly access, read, or monitor private messages, phone calls, photos, videos, files, contacts, passwords, activity inside other apps, or other users’ accounts or devices unless the user gives clear permission and the access is required for a specific feature provided by the application.

AI‑generated suggestions may not always be accurate. Users should review important recommendations before taking action.

## 4. App Monitoring and Management

The application may allow users to add, connect, manage, or monitor apps and services linked to their own account.

Monitoring and management features may include viewing apps or services added by the user, checking connection or security status, displaying authentication or account protection status, sending security‑related notifications, and helping users manage permissions, access, or security settings.

The application does not secretly monitor all applications on the user’s device.

The application only manages or monitors apps, services, or account information that the user connects, authorizes, or adds to the application.

## 5. Face ID and Biometric Authentication

The application may support Face ID, biometric authentication, or device‑based authentication to help users unlock their account or access protected features securely.

Biometric data, including Face ID or fingerprint data, is processed by the user’s device operating system.

This application does not collect, store, transmit, or directly access biometric data.

The application only receives a confirmation from the device indicating whether authentication was successful or failed.

## 6. Two‑Factor Authentication

The application may support two‑factor authentication to provide an additional layer of account protection.

Two‑factor authentication may be used to verify the user’s identity, protect account login, prevent unauthorized access, secure sensitive account actions, and confirm changes to important security settings.

Depending on the feature, two‑factor authentication may use verification codes, email confirmation, authenticator apps, device confirmation, or other secure verification methods.

We do not use SMS‑based codes or phone number verification. Two‑factor authentication in this application relies on email, authenticator apps, or hardware/device confirmation to maintain a high level of security.

## 7. Google User Data

If this application accesses Google user data, it will only access, use, store, or share that data as necessary to provide features clearly disclosed to the user.

Google user data is used only for authorized application functionality, such as sign‑in, account identification, account security, AI assistance requested by the user, or app management features.

We do not sell Google user data.

We do not use Google user data to monitor users outside the application’s stated functionality.

We do not share Google user data with third parties except when necessary to provide or secure the application, when required by law, with service providers that process data on our behalf under confidentiality and security obligations, or with the user’s consent.

## 8. Data Storage and Security

User data may be stored securely using Firebase services, cloud infrastructure, or our application database. We implement high‑level protection measures, including encryption at rest and in transit, to safeguard user information.

We apply reasonable technical and organizational safeguards designed to protect user data against unauthorized access, loss, misuse, alteration, or disclosure.

Security measures may include encrypted communication where supported, access controls, authentication protections, secure database rules, monitoring for suspicious access, and restricted administrative access.

No method of electronic storage or transmission is completely secure, but we work to protect user information using reasonable security practices.

## 9. Data Retention

We retain user data only for as long as necessary to provide the application’s features, maintain account security, resolve technical issues, comply with legal obligations, enforce our terms, or protect against misuse.

When data is no longer needed, we may delete it, anonymize it, or securely retain it where required by law.
