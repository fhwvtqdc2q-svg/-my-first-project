# Security Policy

## الإبلاغ عن الثغرات الأمنية

إذا وجدت مشكلة أمنية في هذا المشروع، يرجى عدم فتح Issue علني يحتوي على تفاصيل حساسة أو معلومات قابلة للاستغلال.

يرجى التواصل مع مالك المستودع مباشرة، أو فتح بلاغ مختصر لا يتضمن مفاتيح API، رموز وصول، كلمات مرور، بيانات اعتماد، أو تفاصيل تقنية قد تسمح باستغلال المشكلة.

## المبادئ الأمنية

- لا يتم تخزين مفاتيح API أو رموز الوصول داخل المستودع.
- لا يتم رفع ملفات `.env` أو أي ملفات تحتوي على بيانات اعتماد حقيقية.
- لا تتم إضافة اتصالات خارجية إلا عند وجود حاجة وظيفية واضحة ومبررة.
- تتم مراجعة التبعيات والمكتبات قبل إضافتها إلى المشروع.
- يتم اعتماد مبدأ الحد الأدنى من الصلاحيات عند إضافة أي إعدادات أو وظائف مستقبلية.
- يتم توثيق أي آلية تتبع أو اتصال خارجي بشكل واضح قبل استخدامها.

## نطاق السياسة الحالي

المشروع حاليًا في مرحلة الإعداد الأولي، ولا يحتوي على كود تشغيلي أو تبعيات. سيتم تحديث هذه السياسة عند إضافة وظائف أو تقنيات جديدة.
